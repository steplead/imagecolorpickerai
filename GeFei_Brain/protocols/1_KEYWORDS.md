# Protocol 1: Keyword Engineering (The "Ge Fei" Standard)

> **Core Rule:** "For a new site, specificity is survival."

## 1. Selection Criteria (The Metrics)
| Metric | New Site (0-3 Months) | Established Site (>6 Months) |
| :--- | :--- | :--- |
| **KD (Difficulty)** | **< 30** (Ideally < 15) | < 60 |
| **Search Volume** | **600 - 800** / month | > 2000 / month |
| **CPC** | > $0.01 (Must have commercial value) | > $0.50 |
| **Intent** | Informational ("How to"), Tool ("Calculator") | Commercial ("Best X") |

## 2. The "Suffix Strategy" (Tool Building)
Don't generic keywords. Append these **Product Suffixes** to find gold mines:
-   `Calculator` (e.g., BMI Calculator, ROI Calculator)
-   `Generator` (e.g., Name Generator, Story Generator)
-   `Converter` (e.g., PDF to Word)
-   `Template`, `Checker`, `Simulator`

## 3. Validation Workflow
1.  **Google Trends:** Check "Past 30 Days". Is it rising?
2.  **SERP Check:** Are the top results "Weak"? (Forums, Blogs, or ugly tools).
3.  **KGR (Golden Ratio):** If `Allintitle / Volume < 0.25`, it is a guaranteed win.

## 4. Anti-Rules (The "Don'ts")
*   ❌ **Do NOT** target "Brand Names" (e.g., "ChatGPT") directly.
*   ❌ **Do NOT** use `meta keywords` tag.
*   ❌ **Do NOT** target high-KD words just because volume is high.
